# Project 2 CPU Reoptimized Summary

## What Changed

- C++ backend `gmp_backend.cpp` 新增 `leaf_terms` 小块迭代路径，避免递归一直下探到单项。
- chunk merge tree 改成按轮次 OpenMP 并行归并，不再把顶层大乘法串行做完。
- 重新调参后，高位数最佳 `chunk_terms` 从旧的 `524288~1048576` 明显前移到 `131072~262144`。

## Takeaways

- 这轮优化不是只改善小规模。`100M` 从旧的 `65.56s` 压到 `47.44s`，`150M` 和 `200M` 也分别比旧 C++ 基线快约 `38%` 和 `44%`。
- 优化后的纯 CPU 在 `150M~200M` 维持在约 `1.8e6 digits/s`，已经明显高于 `bounded-stream-auto` 的 `1.42e6~1.45e6 digits/s`。
- 因此当前工程上更合理的选择是：把优化后的 `C++ + GMP + OpenMP` 作为默认高位数主线，GPU hybrid 保留为更高吞吐的探索路线，而不是继续作为默认路线。
